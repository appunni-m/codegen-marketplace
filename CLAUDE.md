# CLAUDE.md — AI Plugin Marketplace

Author plugins once, ship to **Codex, Claude Code, Cursor, Gemini CLI, Kiro,
and Vercel Skills CLI**. Pi consumes selected plugin directories directly.
This repo is a thin consumer of the
[`@ai-plugin-marketplace`](https://www.npmjs.com/package/@ai-plugin-marketplace/cli)
toolkit — all build, scaffold, and validation logic lives in those packages.

Skills follow the **[Agent Skills](https://agentskills.io) open specification**.
The spec section below is canonical for this repo. For *how* to write, evaluate,
and iterate on skills, invoke the installed skills (see [Delegation](#delegation)).

---

## Agent Skills spec (canonical for this repo)

A skill is a **directory** containing at minimum a `SKILL.md` file. The directory
name must match the `name` field exactly.

### Directory structure

```
skill-name/
├── SKILL.md          # Required: YAML frontmatter + Markdown instructions
├── scripts/          # Optional: executable code
├── references/       # Optional: docs loaded on demand as context
├── assets/           # Optional: files used in output (templates, images, fonts)
└── examples/         # Optional: working, runnable examples
```

### SKILL.md frontmatter

| Field           | Required | Constraints |
| --------------- | -------- | ----------- |
| `name`          | **Yes**  | Max 64 chars. Lowercase letters, numbers, hyphens only. Must not start/end with hyphen. No consecutive hyphens. **Must match the parent directory name.** |
| `description`   | **Yes**  | Max 1024 chars. Non-empty. Third person with specific trigger phrases in quotes: `"create X"`, `"configure Y"`. This is the *only* field agents read at discovery. |
| `license`       | No       | License name or reference to a bundled license file. |
| `compatibility` | No       | Max 500 chars. Environment requirements. Most skills don't need this. |
| `metadata`      | No       | Arbitrary key-value map (`author`, `version`, etc.). |
| `allowed-tools` | No       | Space-separated pre-approved tools. Experimental. |

### Progressive disclosure

1. **Discovery** — Only `name` + `description` loaded at startup (~100 tokens)
2. **Activation** — Full `SKILL.md` body loads when task matches description. Keep under **5,000 words**; target 1,500–2,000.
3. **Resources** — Files in `scripts/`, `references/`, `assets/`, `examples/` load only when the agent decides it needs them

### Writing style (mandatory)

**Body:** Imperative/infinitive form. Verb-first, objective. Never "you" or "your."

```markdown
# Correct
To validate the configuration, run `scripts/check.sh`.
Parse the frontmatter and extract the version field.

# Incorrect
You should validate the configuration.
You can parse the frontmatter to extract the version.
```

**Description:** Third person with quoted trigger phrases.

```yaml
# Correct
description: This skill should be used when the user asks to "rotate a PDF",
  "extract PDF pages", "merge PDF files", or needs to manipulate PDF documents.

# Incorrect
description: Use this skill for PDF tasks.
description: Helps with PDFs.
```

### File references

Use relative paths from the skill root. Tell the agent *when* to load each file:

```markdown
Read `references/api-errors.md` if the API returns a non-200 status code.
Run `scripts/validate.py --input results.json` to check the output.
```

### Validation

```bash
pnpm check:skills
```

### Final review before committing a skill

Run `pnpm check:skills`, inspect trigger overlap with neighboring skills, and
forward-test any workflow that executes commands or transforms files. Add a
deterministic test for bundled scripts. Use `skill-creator` for additional
trigger or quality evaluation when the change is broad enough to justify it.

## Marketplace workflow

### Add a skill

```bash
# 1. Scaffold the plugin (creates the full skeleton — plugin.json, aipm.config.ts,
#    starter SKILL.md, GEMINI.md, POWER.md, etc. Don't recreate what scaffold emits.)
pnpm scaffold <plugin-name>

# 2. Edit the generated SKILL.md at plugins/<plugin-name>/skills/<plugin-name>/SKILL.md
#    Add optional scripts/, references/, assets/, examples/ only if needed

# 3. Build and validate
pnpm build     # regenerates all toolkit-owned artifacts
pnpm check     # schema + consistency + freshness checks
```

To add a **second skill** to an existing plugin, create only the skill
subdirectory — scaffold already did the rest:

```bash
mkdir -p plugins/<plugin-name>/skills/<new-skill-name>
# Write SKILL.md + optional supporting directories
```

`aipm build` must produce a clean git tree — CI enforces this. Commit both
authored sources **and** generated output.

### Key constraint: generated vs authored

These paths at the **repo root** are GENERATED by `aipm build`:

```
skills/  commands/  agents/  hooks/  steering/
GEMINI.md  POWER.md  gemini-extension.json  .kiro/  dist/
.claude-plugin/marketplace.json  .cursor-plugin/marketplace.json
```

**Never edit them directly.** Edit sources under `plugins/<name>/`, then run
`aipm build`. The toolkit owns these paths (tracked in `.aipm/generated-root.json`).

### Repository structure

```
.
├── aipm.workspace.ts              # Marketplace metadata
├── plugins/
│   └── <plugin-name>/
│       ├── aipm.config.ts          # targets + version
│       ├── .claude-plugin/plugin.json
│       ├── skills/                 # ← SKILL.md files go here
│       ├── commands/  agents/  hooks/  rules/
├── skills/  commands/  agents/  hooks/  steering/  # GENERATED
├── GEMINI.md  POWER.md  gemini-extension.json  .kiro/  # GENERATED
├── .claude-plugin/marketplace.json   # GENERATED
├── .cursor-plugin/marketplace.json   # GENERATED
└── .aipm/generated-root.json
```

### Targets (`plugins/<name>/aipm.config.ts`)

```ts
import { defineConfig } from '@ai-plugin-marketplace/core';
export default defineConfig({
  version: '0.2.0',
  targets: ['claude', 'codex', 'cursor', 'gemini', 'kiro', 'vercel'],
});
```

| Target | Emits |
| ------ | ----- |
| claude | `.claude-plugin/plugin.json`, hooks, skills |
| codex  | `.codex-plugin/plugin.json` |
| cursor | `.cursor-plugin/plugin.json` |
| gemini | `GEMINI.md`, `gemini-extension.json` |
| kiro   | `POWER.md`, `.kiro/`, `steering/` |
| vercel | Vercel Skills CLI manifests |
| open-plugins | `.plugin/plugin.json`, root `marketplace.json` |

The scaffold output is authoritative — `aipm scaffold` emits the full set of
targets the toolkit supports. Trust it over this table if they disagree.

### Coverage MCP sync

When Coverage MCP changes MCP tool behavior, input/output schema, tool
descriptions, server instructions, connector guidance, or the agent workflow,
update the marketplace in the same change window so installed agents do not
learn stale behavior.

Authoritative source files for the `testing` plugin are:

- `plugins/testing/README.md` for marketplace/user-facing connector guidance.
- `plugins/testing/skills/use-coverage-mcp/SKILL.md` for the agent workflow.
- `plugins/testing/compatibility.json` and
  `plugins/testing/scripts/install-pi-mcp.mjs` when connector invocation or
  source URL changes.
- Root `README.md` when examples or installation guidance change.

Do not edit generated root artifacts to patch Coverage MCP guidance. Edit the
plugin source files above, then run the required marketplace build/check
workflow if generated outputs are affected. After publishing or pushing, refresh
the local Codex install with `codex plugin add testing@codegen-marketplace` and
verify the cached plugin under `~/.codex/plugins/cache/codegen-marketplace/testing/`.

### Constraints

- **Single-artifact hosts** - Gemini and Kiro each have one repository-level
  artifact. Only one plugin may declare each of those targets. Other plugins
  may still target registry-based hosts.
- **Generated ownership** - Never hand-edit root registries, root host
  artifacts, generated `skills/`, or `dist/`.
- **Skill `name` must match directory name** — enforced by spec and `skills-ref validate`.
- **Skills are portable** — avoid host-specific assumptions. Works across 30+ agents.
- **Use kebab-case** for all file and directory names.
- **File naming:** commands/agents are `.md` files, skills are subdirectories with `SKILL.md`.
- **External content** - Do not copy a third-party skill without a compatible
  license, preserved notices, and attribution.
